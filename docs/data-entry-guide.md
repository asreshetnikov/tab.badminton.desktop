# Порядок ввода данных при проведении турнира

Документ описывает рекомендуемую последовательность работы в приложении — от подготовки до финала. Следование этому порядку позволяет избежать ошибок: данные каждого следующего шага опираются на уже введённые.

---

## Фаза 1 — Подготовка (за несколько дней до турнира)

### Шаг 1. Создать площадку (Venue)

*Sidebar → Players → раздел Venues или при создании турнира*

- Название зала / спортивного комплекса
- Адрес

> Площадку достаточно создать один раз — она переиспользуется в следующих турнирах.

---

### Шаг 2. Создать турнир

*Dashboard → New Tournament*

Обязательные поля:
- **Название** турнира
- **Дата начала** и **дата окончания**
- **Площадка** (выбрать из созданных)

После сохранения открывается карточка турнира — отправная точка для всех дальнейших действий.

---

### Шаг 3. Добавить корты

*Карточка турнира → Edit → раздел Courts*

Для каждого корта — только название (Court 1, Court 2, …). Количество кортов определяет ёмкость расписания.

---

### Шаг 4. Задать временные настройки по дням

*Карточка турнира → Edit → раздел Day Settings*

Для каждого дня турнира:
- **Время начала** игрового дня (например, 09:00)
- **Продолжительность одного матча** по умолчанию (например, 30 мин)

Эти значения используются при авто-расписании. Если турнир однодневный — настраивается один день.

---

### Шаг 5. Добавить события (Events / категории)

*Карточка турнира → раздел Events → Add*

Для каждой соревновательной категории:
- **Категория**: MS / WS / MD / WD / XD
- **Название** (заполняется автоматически, можно изменить; пример: «MS U19»)
- **Возрастное ограничение** (опционально): U{возраст} или {возраст}+
- **Лимит участников** (опционально)

> Порядок событий можно менять перетаскиванием.

---

### Шаг 6. Добавить игроков в базу

*Sidebar → Players → Add Player*

Для каждого игрока:
- Имя и фамилия
- Пол (M / F) — определяет категории, в которых игрок может участвовать
- Год рождения (опционально, нужен для возрастной валидации)
- Клуб (опционально)

> Игроков достаточно создать один раз. В следующих турнирах они уже будут в базе.

---

### Шаг 7. Зарегистрировать участников на турнир

*Карточка турнира → Registrations → Add*

Выбрать игроков из базы и добавить в список заявок турнира. Каждая заявка имеет статус:
- **pending** — подана, ожидает подтверждения
- **accepted** — принята; при принятии автоматически создаётся singles-команда (MS или WS)
- **rejected** — отклонена

**Принять заявку** нужно явно — без этого игрок не попадёт в команды и не сможет участвовать в сетках.

---

### Шаг 8. Создать пары для парных разрядов

*Карточка турнира → Teams → Add Team*

Для MD, WD, XD:
- Создать команду вручную
- Привязать двух игроков
- Указать категорию команды
- Зарегистрировать команду в нужном событии турнира

> Одиночные команды (MS/WS) создаются автоматически при принятии заявки игрока — добавлять вручную не нужно.

---

### Шаг 9. Создать этапы (Rounds) и добавить участников

*Карточка турнира → Rounds*

Для каждого события создать один или несколько этапов в нужном порядке:

| Тип | Когда использовать |
|-----|-------------------|
| **Round Robin** | Групповой этап: все играют со всеми |
| **Playoff** | Сетка на выбывание |

Для каждого этапа:
1. Нажать **Add Round**, задать название и тип
2. Флаг **All Entries** автоматически добавляет всех участников события в этап
3. Если нужно добавить только часть — снять флаг и добавить вручную через кнопку **Add Teams** в развёрнутом этапе

**Типичная схема «группы → плей-офф»:**
1. Round Robin «Groups» — все участники события
2. Playoff «Main Draw» — победители групп (добавляются вручную после завершения групп)

---

### Шаг 10. Сгенерировать матчи

*Карточка турнира → Rounds → кнопка Matches у нужного этапа → GroupsView*

- **Round Robin**: нажать **Generate Matches** — система строит расписание по алгоритму Бергера (все пары, без повторов)
- **Playoff**: нажать **Generate Bracket** — система строит сетку на выбывание по числу участников

> Генерацию нельзя отменить, если уже введены результаты. Убедитесь, что состав участников финальный перед генерацией.

---

## Фаза 2 — День турнира (оперативная работа)

### Шаг 11. Составить расписание

*Карточка турнира → Schedule*

**Ручной режим:**
- Слева — неназначенные матчи (сгруппированы по туру/раунду сетки)
- Перетащить матч в колонку правой части (по кортам и датам) на нужный слот
- Система предупредит о конфликте, если игрок уже занят в другом матче в то же время

**Авто-режим:**
- Кнопка **Auto-schedule** — расставляет все неназначенные матчи автоматически с учётом приоритетов и временных настроек

**Дополнительные настройки расписания** (кнопка Settings в Schedule):
- Продолжительность матчей по стадии сетки (четвертьфинал дольше группы)
- Per-day время начала и длительность матча

---

### Шаг 12. Вводить результаты матчей

*Карточка турнира → Rounds → Matches → GroupsView*

Для каждого сыгранного матча:
1. Кликнуть на матч в списке или сетке
2. Ввести счёт по сетам (S1, S2, S3 при необходимости)
3. Сохранить

Автоматически после сохранения:
- **Round Robin**: обновляется таблица группы (wins, sets, points)
- **Playoff**: победитель продвигается в следующий матч сетки

Доступные статусы матча: `scheduled` → `in_progress` → `finished` / `walkover` / `retired`

---

### Шаг 13. Контролировать ход группового этапа

*GroupsView → таблица Standings*

Таблица обновляется в реальном времени после каждого результата. Критерии расстановки:
1. Победы
2. Разница сетов
3. Разница очков

По завершении всех матчей группы — вручную перенести нужных участников в следующий этап (Playoff).

---

### Шаг 14. Вести плей-офф до финала

*GroupsView → кнопка View Bracket (для playoff-этапа)*

SVG-сетка отображает текущее состояние турнира. После ввода результата матча победитель автоматически появляется в следующем матче.

Следить за:
- Матчами со статусом `scheduled` — они ещё не сыграны
- Матчами без назначенного времени — стоит добавить в расписание

---

## Типичные ошибки и как их избежать

| Ситуация | Что случится | Решение |
|----------|-------------|---------|
| Добавили игрока в этап, не приняв заявку | Матчи будут сгенерированы, но игрок не числится принятым | Сначала принять заявку (статус → accepted), потом добавлять в этап |
| Сгенерировали матчи до финального состава | Придётся удалять и пересоздавать этап | Сформировать состав участников этапа полностью до генерации |
| Не задали временные настройки дней | Авто-расписание использует дефолты (могут не совпадать с реальностью) | Задать start_time и match_duration в Edit турнира до составления расписания |
| Назначили матч на то же время другому игроку | Конфликт: система покажет предупреждение | Проверять предупреждения в Schedule или использовать Auto-schedule |

---

## Краткая схема-шпаргалка

```
[Venue] → [Tournament] → [Courts] → [Day Settings]
    ↓
[Events/Categories]
    ↓
[Players] → [Registrations: pending → accepted]
    ↓
[Teams] (пары — вручную, одиночки — автоматически)
    ↓
[Rounds: Round Robin → Playoff] → [Add Participants]
    ↓
[Generate Matches / Bracket]
    ↓
[Schedule: Auto или Drag & Drop]
    ↓
[Enter Results] → [Standings / Bracket advances]
    ↓
[Repeat for next stage]
```
