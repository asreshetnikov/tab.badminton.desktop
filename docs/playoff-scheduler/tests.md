Базовые тесты

1. Планирует будущие playoff-матчи без известных участников
   Сетка на 4 или 8 команд: первые матчи ready, полуфинал/финал без team1_id/team2_id. После autoSchedule у всех незавершённых матчей должен быть scheduled_at.
2. Не трогает завершённые матчи
   Матч со статусом finished, walkover или retired и уже заданным scheduled_at должен сохранить своё время.
3. Соблюдает зависимости дерева
   Родительский матч должен начинаться не раньше:
   max(child.scheduled_at + child_duration) + rest_minutes.
   Это главный инвариант предварительного расписания.
4. Планирует уровни сетки по порядку
   Для playoff-дерева: все матчи bracketRound=2 должны быть позже зависимых матчей bracketRound=1, финал позже полуфиналов.

Длительности и настройки
5. Использует duration по стадии
   Если tournament_stage_durations задаёт разные длительности для bracket_round=1 и 2, parent notBefore должен считаться от длительности child-раунда, а занятость корта — от длительности текущего матча.

6. Использует day settings как fallback
   Если stage duration не задана, используется tournament_day_settings.match_duration.
7. Использует дефолт 60 минут
   Если нет ни stage duration, ни day setting, расписание должно строиться с шагом 60 минут.
8. Учитывает rest_minutes турнира
   При rest_minutes = 0, 30, 90 parent-матч должен сдвигаться соответственно.

Корты и slot-fill
9. Выбирает самый ранний доступный корт
   На двух кортах будущие матчи должны распределяться так, чтобы следующий матч вставал на корт с минимальным возможным max(court.endTime, notBefore).

10. Не простаивает, если другой корт свободен раньше
    Сценарий: один корт занят длинным матчем, второй свободен. Следующий независимый будущий матч должен попасть на второй корт по времени.
11. Продолжает timeline после round robin и ready playoff
    Если перед _preScheduleBracket уже запланированы RR/ready playoff матчи, будущие матчи должны стартовать после текущих courtSlots, а не от начала дня.

Несколько категорий
12. Сортирует будущие матчи одного уровня по earliest notBefore
    Если у XD следующий раунд может начаться в 13:30, а у MD в 15:00, XD должен получить более ранний слот, даже если оба матча одного bracketRound.

13. Не смешивает зависимости разных playoff-раундов
    Матч категории A должен зависеть только от своих left_match_id/right_match_id, а не от матчей категории B.

Граничные случаи
14. Нет playoff-матчей для предварительного расписания
    Если все playoff-матчи уже ready и запланированы фазой 2, autoSchedule должен завершиться без ошибок.

15. Пустой список кортов
    autoSchedule должен ничего не делать и не падать.
16. Неполное дерево
    Если у parent есть только один child или child не имеет scheduled_at, алгоритм не должен падать; parent планируется по доступным зависимостям.
17. Повторный запуск идемпотентен
    Два последовательных autoSchedule на одинаковых данных должны давать одинаковые scheduled_at для незавершённых матчей.
18. Учитывает not_before_hard для ready-матчей до pre-schedule
    Это не напрямую _preScheduleBracket, но важно для входных courtSlots: если ready playoff сдвинут через hard constraint, будущие матчи должны планироваться после него.
