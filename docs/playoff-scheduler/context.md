# Контекст проекта: Playoff Scheduler

## Что это

Сервис автоматизации составления расписания для турниров по ракетным видам спорта (бадминтон и др.) с форматом single elimination playoff.

Полное описание задачи: [draft.md](draft.md)  
Алгоритм составления расписания: [algorithm.md](algorithm.md)

---

## Суть задачи

Игрок в одном турнире может участвовать в трёх категориях: singles, doubles, mixed-doubles. Все категории могут идти параллельно в один день. Нужно автоматически строить и актуализировать расписание матчей по мере продвижения игроков по сетке.

---

## Ключевые ограничения

- **N площадок** — не более N матчей одновременно
- **Single elimination** — участники следующего матча известны только после завершения предыдущих
- **Отдых M минут** между последовательными матчами одного игрока — soft constraint (предупреждение, не блокировка)
- **not_before_hard** — жёсткое ограничение на время начала, задаётся вручную (в т.ч. для управления игровыми днями)
- **not_before_soft** — вычисляется автоматически из времени окончания последнего матча + M
- **effective_not_before = max(soft, hard)**
- В doubles/mixed: effective_not_before матча = max по всем участникам

---

## Алгоритм (кратко)

Очередь матчей сортируется по приоритету. При освобождении площадки берётся верхний доступный матч.

**Приоритет матча** = `category_depth` (раундов до финала) + `max(cross_pending)` по участникам (несыгранные матчи в других категориях).

Очередь пересчитывается при каждом событии: старт турнира, внесение результата, ручное изменение расписания.
